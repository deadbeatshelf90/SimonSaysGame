Issues with the plan to use LED’s:

Originally, I intended to use 4 colour coded LEDs to signify the different possible elements of the sequence. However, I found that the LEDs would not shine brightly if their ground pin was being used as a digital read pin by the Micro-Bit. To combat this issue, I removed the LED’s and used the Micro-Bit's display screen as my digital output and instead of having colours show the elements of the sequence, I used directions (up, down, left and right). This meant that I needed to rewrite some of the game code too as it no longer needed to digitally write to the bread board to turn on the LEDs and the buttons were then rearranged to show the 4 directions.

My important code:

One important piece of code I created was the code that automatically counts the children of the branch sessions by first checking is the reference to the database empty by using the None type. If the reference is empty then the code sets the child of the main branch to ‘sessions’: ‘’ and the variable next_session to 1, either way the code then makes a new database reference to be the child of ‘sessions’. Once the new reference has been made the code takes all the data from the branch and makes a variable called last_element_key that turns the branch data into a list of keys and finds the last key in the list. The variable next_session is then set to the last key and from there is set to the number at the end of the list, e.g. 2 if the key is session 2. The variable next_session is then added to by 1 to make the number of the next session. If there is no data in the database then next_session is set to 1. When the code pushes the validated data to the database it checks to see if the Result of the round was a loose, if so then next_session is added to by 1 again as the current session has ended.
