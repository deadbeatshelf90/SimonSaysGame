Plan and Design

This project will operate much like the well- known game Simon Says. The system will pick random directions from an array, the number of directions increasing with each round. Using a pulse sensor, the system would monitor the users hear rate through the duration of each round. The heartrate data and the results of the round will be stored in firebase and later used by the analysis component and the what if questions.

I decided to incorporate some features of Universal Design such as Equitable Use, Simple and Intuitive Use and Low Physical Effort.

I will use Agile to develop this project as it takes form as stages of development such as sprint planning (e.g. stages for microbit, python, database etc), acceptance criteria (what needs to be done in each stage), kanban (work will be planned in order of priority) and acceptance testing (verify that each stage meets the requirements as set out by the brief)

To start, the user presses the A button on the "Game Micro Bit" to initiate the game. The system compares the user's sequence with its own and notifies the user if they've won the round. Upon winning, the round counter increments until reaching round 6, at which point the system informs the user, "That's enough for today." Results, along with heart rate data, are sent to Firebase at the end of each round. This data aims to provide users with insights into their well-being going forward.

The model's inputs include a heart rate monitor as the analogue input and 4 buttons for user sequences as digital inputs. Outputs consist of the Micro:Bit LED screen displaying the sequence to replicate as the digital output, and the in-built buzzer providing feedback for each sequence element, with each direction having its respective tone as the analogue output.

The technologies that I used are as followed:

  • I will use a Micro-Bit for my embedded system.
  • I will use the Thonny Ide to write my python code.
  • I will use the website MadeCode to program the Micro-Bit.
  • I will use the Ide Visual Studio Code to wrie my html and JavaScript code.
  • I will use the Chart.js to plot my graphs.
  • I will use Obs Studio to record my website functionality.
  • I will use OpenShot Video Editor to create the video Meeting the Brief.

I plan to model stress levels using the heartrate sensor to predict the user’s success chances in the game, all to help them reduce their stress levels while playing.

My two ‘What-if’ questions that I will aim to answer will be “If the number of rounds were to be reduced to x, what would be the chance of a success?” and “What if the user was stressed, would this support or contradict the conclusion given?” These questions are aimed to make the user’s goals using the device realistic as to help them reduce stress.

The main code of my project will be created in Python and JavaScript, with the presentation of my brief being created in HTML and designed with CSS. The data will be sent to Google Firebase to cleanly store the data for further use.
